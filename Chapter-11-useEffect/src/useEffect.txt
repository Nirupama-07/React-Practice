What is a Cleanup Function?

A cleanup function is a function that React calls before an effect runs again or when the component is removed (unmounted).

The syntax is:

useEffect(() => {
  console.log("Effect Started");

  return () => {
    console.log("Cleanup");
  };
}, []);

Notice the return.

That returned function is called the cleanup function.

Why do we need cleanup?

Imagine you have a timer.

setInterval(() => {
  console.log("Hello");
}, 1000);

This prints:

Hello
Hello
Hello
Hello
...

every second.

Now imagine your component disappears from the screen.

Should the timer keep running?

❌ No.

If it keeps running:

CPU is wasted.
Memory is wasted.
Bugs can happen.

So before React removes the component, it says:

"Clean up whatever you started."

That's exactly what the cleanup function is for.

(The Golden Rule of useEffect)

Keep this in mind:

State changes
      ↓
Component re-renders
      ↓
React updates the DOM
      ↓
Cleanup previous effect (if needed)(When the component unmounts, React only runs the cleanup. There is no new effect, because the component is gone.)
      ↓
Run the new effect

That's the lifecycle whenever an effect with dependencies runs again.
